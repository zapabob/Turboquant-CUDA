### 実装詳細: `build_best_per_layer_selector`

- 各 layer × bit_setting グループで `hidden_cosine_similarity` の平均が最大の Triality view を選択

- 出力: manifest DataFrame + selected_frame DataFrame

### 実装詳細: `compute_triality_selector_statistics`

- `best_per_layer_hidden_cosine` ポリシー vs `key_only_block_so8_learned` のペア Wilcoxon 検定
- 評価メトリクス: `hidden_cosine_similarity`（alternative="greater"）, `next_logit_kl`（alternative="less"）
- Holm-Bonferroni 多重比較補正
- `promotion_gate` 行: hidden cosine 有意 + KL 非悪化 + memory ratio 変化なしで `True`

### 設計構造

#### Track 1: Multiscreen KV スコアリング（独立実装可能）

```
新規ファイル:
  turboquant/research_extension/multiscreen_kv.py
      - normalize_unit()
      - build_mipe_factor()   ← MiPE (Mixed Positional Encoding)
      - trim_and_square()     ← スパース関連性ゲート (alpha = clamp(1 - r*(1-s), 0)^2)
      - compute_k_relevance() ← [batch, heads, k_len] relevance スコアを返す
      - multiscreen_relevance_topk_indices()

変更ファイル:
  turboquant/allocation.py
      + from_multiscreen_relevance(regular_bits, outlier_bits, outlier_count) classmethod
      + make_bitwidths_from_relevance(relevance) → [k_len] bitwidths tensor
  turboquant/research_extension/__init__.py
      + compute_k_relevance, multiscreen_relevance_topk_indices を export

新規スクリプト:
  scripts/research_validate_multiscreen_kv.py
      比較モード: exact / key_only_random / key_only_block_so8_triality_vector / multiscreen_relevance
```

**移植元対応:**

| turboquant 側 | multiscreen-pytorch 側 |
|---|---|
| `normalize_unit` | `layers.py:15` |
| `build_mipe_factor` | `layers.py:36–57` |
| `trim_and_square` | `layers.py:60–70` |
| `compute_k_relevance` | `layers.py:96–138` の `compute_similarity` + forward |

**重要な設計決定:** `from_multiscreen_relevance` は `relevance` テンソルを受け取らない（コンストラクタでは不要）。`relevance` は `make_bitwidths_from_relevance(relevance)` に渡す。
