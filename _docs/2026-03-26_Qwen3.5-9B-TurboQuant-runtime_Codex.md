## Code fixes made during runtime bring-up

### 1. Repo id / path handling

File:
- `H:\Qwen3.5-9B-SOT-Deployment\qwen35_rtx3060\integration\qwen35_loader.py`

Change:
- Preserved Hugging Face repo ids like `Qwen/Qwen3.5-9B` as strings instead of forcing `Path(...)`.

### 2. Rotation dtype/device alignment

File:
- `H:\Qwen3.5-9B-SOT-Deployment\qwen35_rtx3060\turboquant\rotation.py`

Issue:
- TurboQuant rotation used a float32 CPU-oriented matrix path that failed on Qwen KV tensors in `bfloat16`.

Change:
- `rotate()` and `unrotate()` now align the rotation matrix to the input tensor `device` and `dtype` before matmul.

### 3. QJL device/dtype alignment

Files:
- `H:\Qwen3.5-9B-SOT-Deployment\qwen35_rtx3060\turboquant\qjl.py`
- `H:\Qwen3.5-9B-SOT-Deployment\qwen35_rtx3060\turboquant\quantizer_prod.py`

Issue:
- The QJL projection matrix remained on CPU while runtime KV residuals lived on CUDA.

Change:
- `QJLTransform.quantize()` and `QJLTransform.dequantize()` now align the projection matrix to the active tensor `device` and `dtype`.
- `decode_for_inner_product()` now does the same for the inner-product path.
