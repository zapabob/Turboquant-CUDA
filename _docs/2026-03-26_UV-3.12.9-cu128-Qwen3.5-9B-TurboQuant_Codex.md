## 実装目的

- Windows ネイティブ環境で `uv` を正にした Python 3.12.9 の実行基盤へ移行する
- PyTorch `cu128` を使う前提で Hugging Face Qwen3.5-9B / Qwen3.5-9B-Base の TurboQuant 実験導線を整える
- TurboQuant の適用対象を KV cache に限定し、重み量子化は `bitsandbytes` の補助戦略として扱う

## 変更概要

### 1. 環境管理を `uv` 前提へ移行

- `.python-version` を追加して `3.12.9` を固定
- `pyproject.toml` の `requires-python` を `>=3.12,<3.13` に変更
- `pyproject.toml` に `cu128` / `dev` optional extras を追加
- `tool.uv.sources` と `[[tool.uv.index]]` を使って `torch` を `https://download.pytorch.org/whl/cu128` から引く構成へ変更
- `uv.lock` を生成
- `requirements.txt` は legacy fallback 扱いとし、canonical setup は `uv` であることを明記

### 2. ドキュメントと作業規約を更新

- `README.md` を `uv python install 3.12.9` / `uv venv --python 3.12.9` / `uv sync --extra cu128 --extra dev` ベースに更新
- `README.md` に Qwen3.5-9B と Qwen3.5-9B-Base の smoke 実行例を追加
- `AGENTS.md` に `uv` と Python 3.12.x を canonical path として追記
- `scripts/bootstrap_uv.ps1` を追加し、Windows PowerShell から `uv` セットアップを 1 本化

### 3. ランタイム前提をコードに反映

- `turboquant/runtime.py` を追加
- 以下のランタイム定数を導入
  - `REQUIRED_CUDA = "12.8"`
  - `DEFAULT_MODEL_ID = "Qwen/Qwen3.5-9B"`
  - `BASE_MODEL_ID = "Qwen/Qwen3.5-9B-Base"`
- Python 3.12 以外を明示的に弾く `require_supported_python()` を追加
- CUDA バージョンと可視 GPU をチェックする `cuda_matches_target()` を追加

### 4. スクリプトを Qwen + cu128 前提へ更新

#### `scripts/env_check.py`

- Python exact version を出力
- `python_supported`
- `torch.version.cuda`
- `target_cuda`
- `target_cuda_match`
- default model / default weight load を出力
- Python 3.12.x ではない場合、または CUDA `12.8` でない場合は failure に積むように変更

#### `scripts/capture_qwen_kv.py`

- default model を `Qwen/Qwen3.5-9B` に維持
- `--model-preset default|base` を追加
- `--weight-load 4bit|8bit|none` を追加
- smoke 向けの `--max-length` デフォルトを `96` に変更
- 実行前に Python 3.12 と CUDA `12.8` + GPU 可視性を強制チェック
- `bitsandbytes` 4bit を標準ルートに変更

#### `scripts/validate_attention_scores.py`

- `--model-id` を追加
- `--query-source synthetic|captured` を追加
- capture manifest があれば `model_id` を metrics に反映
- 現時点では captured query replay は未実装で、`captured` 指定時は明示的に `NotImplementedError` を出す

### 5. 補助整備

- `.gitignore` を追加
  - `.venv/`
  - `.venv-*/`
  - `.pytest_cache/`
  - `__pycache__/`
  - `*.egg-info/`

## 失敗・ブロッカー

### 1. `uv sync --extra cu128 --extra dev` の完全成功は未達

`uv sync` 自体は lock と resolve を進めたが、Windows 上のファイルロックと disk space 制約により完全には終わらなかった。

確認できた代表的な失敗:

- `torch` 展開中にファイル利用中 (`os error 32`)
- `bitsandbytes` 展開中に空き容量不足 (`os error 112`)

### 2. 空き容量不足

`Get-PSDrive` 時点の空き容量:

- `H:` 約 `0.45 GB`
- `C:` 約 `14.08 GB`

今回の project root が `H:` 上にあるため、`.venv` / `.venv-cu128` への wheel 展開で詰まった。

### 3. `uv run` による新環境検証は未完了

以下はコード上は対応済みだが、`.venv` への完全インストール未完了のため最終確認まで到達していない:

- `uv run python scripts/env_check.py`
- `uv run python -m pytest -q`
- `uv run python scripts/capture_qwen_kv.py --model-id Qwen/Qwen3.5-9B --weight-load 4bit --max-length 96`
