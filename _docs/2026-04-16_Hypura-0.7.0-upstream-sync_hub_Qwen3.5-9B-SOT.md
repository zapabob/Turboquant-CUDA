# Hypura 0.7.0 Upstream Sync

## Overview

`zapabob/Hypura` の最新 upstream を確認し、vendored Rust workspace の `hypura` / `hypura-sys` を `0.7.0` へ更新した。
upstream 由来の CLI / runtime bridge / optional API key / progress UI 差分を取り込みつつ、この repo 固有の Windows vendor 解決 (`LLAMA_CPP_DIR`, `HYPURA_LLAMA_CPP_DIR`) と `NUM_JOBS` 安全策は維持した。

## Background / Requirements

- ユーザー要求: `zapabob/hypura` の最新を反映し、セマンティックバージョンを上げる。
- upstream 確認結果:
  - repository: `https://github.com/zapabob/Hypura`
  - HEAD commit: `838b52d71f24a4f7ef2074c8df1487a7137d4fc9`
  - commit date: `2026-04-15 22:49:33 +0900`
  - commit subject: `feat(release): sync llama.cpp and bump hypura to 0.7.0`
- ローカル repo は downstream integration repo なので、upstream 追随と同時に local vendor/build constraints を壊さないことが必須。

## Assumptions / Decisions

- semver は upstream HEAD に合わせて `hypura=0.7.0`, `hypura-sys=0.7.0` に更新した。
- `edition` は local workspace への波及を最小化するため `2021` のまま維持した。
- upstream の `build.rs` 変更は全面置換せず、以下を両立するマージ方針を採用した。
  - 取り込む: `Visual Studio 17 2022` generator 固定
  - 維持する: `LLAMA_CPP_DIR` / `HYPURA_LLAMA_CPP_DIR` 解決、`../vendor` と `../../vendor` fallback、`NUM_JOBS=1` 安全既定、`GGML_CCACHE=OFF`
- verification 継続のため、満杯だった `rust/target` を削除して約 14.2 GB を回収した。build 生成物のみで、source of truth ではないため削除可と判断した。

## Implementation Details

- manifest:
  - `hypura` / `hypura-sys` version を `0.7.0` へ更新
  - `hypura` に `indicatif` を追加
  - `Cargo.lock` を更新
- CLI:
  - `rotation_policy` の既定を `RotationPolicy::TrialityVector` に変更
  - `run` / `bench` / `serve` で `RotationPolicy` を文字列ではなく型付きで引き回すよう変更
  - `run` / `bench` / `optimize` / `serve` に ASCII ベースの progress 表示を追加
- runtime bridge:
  - `LlamaTurboquantCliBridge` を追加
  - GGUF metadata が無い場合でも CLI 側の回転ポリシー / seed / TurboQuant env override を `LLAMA_TURBOQUANT_*` に反映
  - `resolve_runtime_setup()` に bridge 引数を追加し、全 call site を更新
- server:
  - `HYPURA_API_KEY` が設定されているときに Bearer token または `X-API-Key` を要求する middleware を追加
  - hot model switch が `hypura serve` 起動時の TurboQuant mode / config / bridge を再利用するよう変更
  - process-global env 更新を helper 関数に集約
- misc:
  - `turboquant_sidecar.rs` に exact fallback の意図コメントを追加
  - `kv_codec_ffi.rs` の test fixture を `gguf_metadata: None` で追従
  - `nvme_backend.rs` の pattern binding を upstream と一致させた
